# 扩展与审计模块说明

## 模块定位

扩展与审计负责后台的审计留痕能力和平台扩展能力，是后台稳定化、平台化的重要部分。

## 一级目标

- 保证关键操作可追溯
- 支撑插件化与开放能力
- 支撑多语言与第三方接入

## 二级导航建议

### 1. 日志与审计

包含：

- 管理员操作日志
- 登录日志
- 敏感操作日志
- 权限变更日志
- 内容删除日志
- 配置修改日志

### 2. 数据报表

包含：

- 会员增长报表
- 发帖趋势报表
- 板块活跃度报表
- 审核与举报统计
- 搜索行为统计
- 附件存储统计

### 3. 扩展中心

包含：

- 插件列表
- 插件安装页
- 插件配置页
- 插件运行状态
- API 密钥管理
- Webhook 配置
- 第三方登录设置
- 语言包管理

## 典型页面

- 管理员操作日志
- 权限变更日志
- 会员增长报表
- 插件列表
- API 密钥管理

## 设计建议

- 审计页面优先强调检索、筛选、导出
- 报表能力适合作为管理决策辅助，而非首页主入口
- 扩展能力建议与日志能力同放底部区域，减少对日常操作干扰
