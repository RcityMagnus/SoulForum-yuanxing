# 论坛后台左侧导航模块划分建议

## 文档说明

本文档从产品经理视角，对论坛管理员后台左侧导航进行重新分组，目标是让后台结构更符合管理员的实际工作路径，而不是按零散功能堆叠菜单。

建议将左侧导航划分为 `8` 大模块。

## 一、工作台

包含：

- 数据总览
- 待处理事项
- 快捷入口
- 告警提醒

说明：

作为后台首页使用，主要解决“今天先处理什么”的问题，适合放在左侧导航最上方。

## 二、内容治理

包含：

- 主题管理
- 帖子管理
- 回收站
- 审核中心
- 举报处理
- 敏感词

说明：

这一组是论坛后台最核心的日常运营模块，建议把“内容管理 + 审核 + 举报”放在一起，因为它们本质上属于同一条内容治理链路。

## 三、会员治理

包含：

- 会员列表
- 会员搜索
- 会员详情
- 用户组调整
- 禁言
- 封号
- 黑名单
- 处罚记录
- 登录日志
- 发帖记录

说明：

这一组解决“人”的管理问题，应与“内容治理”分开，形成独立的会员治理闭环。

## 四、社区结构

包含：

- 分区管理
- 板块管理
- 板块设置
- 板块排序
- 板块权限
- 板块统计规则

说明：

这一组属于论坛的组织结构层，和内容、会员都不同，建议单独成组。

## 五、权限与安全

包含：

- 用户组管理
- 常规权限
- 权限档案
- 后台管理员
- 角色权限配置
- 菜单权限
- 登录安全
- 验证码
- 二步验证
- IP 限制
- 后台白名单

说明：

这一组解决“谁能做什么、如何保证后台安全”的问题，建议集中管理。

## 六、运营与触达

包含：

- 公告管理
- 站内信
- 邮件模板
- 通知模板
- 发送记录
- 注册设置
- 注册协议
- 保留用户名
- 隐私政策

说明：

这一组负责用户触达、注册转化和运营配置，偏运营支撑能力。

## 七、平台支撑

包含：

- 搜索设置
- SEO 设置
- 附件与媒体
- 缓存管理
- 定时任务
- 系统健康检查
- 错误日志
- 备份恢复

说明：

这一组属于系统运行和平台支撑层，建议放在左侧中下部，不要过度前置。

## 八、扩展与审计

包含：

- 管理员操作日志
- 权限变更日志
- 配置修改日志
- 插件管理
- API 密钥
- Webhook
- 第三方登录
- 语言包

说明：

这一组负责平台扩展能力和审计留痕，适合放在左侧较下方。

## 推荐排序

1. 工作台
2. 内容治理
3. 会员治理
4. 社区结构
5. 权限与安全
6. 运营与触达
7. 平台支撑
8. 扩展与审计

## 结论

相较于直接把大量功能页面平铺在左侧导航中，按以上 `8` 大模块分组更符合论坛管理员后台的使用逻辑，也更利于：

- 左侧导航收敛
- 后续原型扩展
- 菜单权限管理
- 后台信息架构稳定化
